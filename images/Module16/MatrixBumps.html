Define matrix multiplication of an $M \times K$ matrix $A$ by a $K \times N$ matrix $B$ as the product $AB$, which is an $M \times N$ matrix, where

$AB[i, j] = A[i, :] \cdot B[:, j]$

The element $AB_{ij}$ is the dot product of the $i^{th}$ row of $A$ and the $j^{th}$ column of $B$

In [5]:
def matrixmul(A, B):
    M = A.shape[0]
    K = A.shape[1]
    assert(B.shape[0] == K)
    N = B.shape[1]
    AB = np.zeros((M, N))
    for i in range(M):
        for j in range(N):
            AB[i, j] = np.sum(A[i, :]*B[:, j])
    return AB

A spectrogram can be thought of as a matrix! This point of view will be very powerful, and it will help us to implement a variety of algorithms efficiently. The example below is a $2049 \times 431$ matrix, which means it has 2049 non-redundant frequency bins and 431 windows in time.

Bump functions¶

Let's say we want to extract just a small frequency region of the spectrogram. We can take the dot product of a column of a spectrogram with something called a "bump function," which peaks around the frequency bin of interest and falls to zero quickly on either end

$b_{c, \sigma}(x) = e^{-(x-c)^2/\sigma^2}$¶

In [23]:
c = 750
sigma = 50
b = np.exp(-(np.arange(Sdb.shape[0])-c)**2/sigma**2)
plt.plot(b)

We can certainly manually apply the bump function to each column with a loop

for j in range(Sdb.shape[1]):
    y[j] = np.sum(Sdb[:, j]*bump)
plt.plot(y)

But we can also use matrix multiplication to do all of this for us. Since Sdb is $2049 \times 431$, we setup a matrix for the bump that's just a "row matrix," which is $1 \times 2049$, and then we multiply by this matrix on the left. At the end, we "flatten" this back to a 1D array, and we get the exact same result since, by definition, matrix multiplication is just doing dot products

In [28]:
c = 750
sigma = 50
Bump = np.zeros((1, Sdb.shape[0]))
Bump[0, :] = np.exp(-(np.arange(Sdb.shape[0])-c)**2/sigma**2)
y = np.dot(Bump, Sdb).flatten()
plt.plot(y)
